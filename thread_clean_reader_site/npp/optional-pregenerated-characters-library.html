Recommended Character Handout Format

For each included character, package:

  • canonical CHR
  • human-readable HTM
  • optional TXT
  • short markdown overview with:
    • character fantasy
    • what kind of player might enjoy the character
    • what the character is good at
    • what still invites customization

Inclusion Standard

A character should usually meet most of these before inclusion:

  • loads cleanly in GCB
  • exports cleanly
  • feels table-ready
  • has a clear FRH identity
  • has a stable canonical file

Notes

  • versioned and _GM_Fix trails remain valuable in GCB_Tests for engineering guardrails
  • the optional player-facing library should include the clean presentation, not the whole workshop history
  • packet-facing advanced pregens such as Father Elias and Silas Rourke can live here without bloating the core NPP surface
