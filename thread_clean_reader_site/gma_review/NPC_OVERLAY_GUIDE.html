NPC Overlay Guide

Base Rule

Speaker identity is primary. NPC overlay is additive.

So: - Sean_GM means Sean as facilitator / GM voice - Sean_GM [NPC: Mr_Smoke] means Sean is still the speaker, but the line is tagged as a major NPC embodiment

Why This Matters

The GMA must learn both:

  1. Sean's stable GM habits
  2. Sean's specific NPC performance habits

If those get merged too early, the model learns a blur instead of a structure.

Major NPCs

Major NPC overlays mark high-value performance material such as: - Reagan - Mr_Smoke - Kobayashi - Doctor_Hope - Doctor_Mike - Bob_Jones

These matter because they show: - pressure - implication - exposition style - social stakes - table reaction patterns

Minor NPCs

Minor NPCs such as: - train_employee - taxi_driver

are useful but usually lighter-weight. Do not overinterpret them unless the scene makes them important.

Caution

An overlay may not appear on every single line of a long NPC block. Use: - surrounding lines - scene title - review context - overlay report

to interpret longer runs of dialogue.

Overlay Audit Trail

Each session includes npc_overlay_report.tsv. Use it when you need: - timestamp confirmation - persona confirmation - scene-title anchor - match-confidence context
