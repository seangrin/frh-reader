## What This Folder Is For

This folder is for character designs that are useful, flavorful, or instructive, but are not automatically the best first-pass onboarding pregens.

Use it for:

- advanced examples
- edge-case builds
- concept evolution
- alternate versions of existing characters
- strong missing-lane candidates that are still being tuned

Do not assume everything here belongs beside the standard pregens in [Characters](C:\Users\seang\OneDrive\Documents\ChatGPT_FRH\FRH_New_Player_Packet_Beta_v1.3\Characters).

## Recommended Structure

Best current structure:

- root:
  - one master explainer
  - folder-level guidance
  - a few cross-character notes when needed
- subfolder per character:
  - `CHR`
  - `HTM`
  - notes
  - build comparisons
  - concept summaries

This is the cleanest pattern because:

- the root stays readable
- each concept line stays bundled
- new variants do not become a flat file pile
- a human can understand the folder at a glance

## One Master Document Or Separate Files?

Best answer:

- keep **one master document** at the root
- keep **one small readme or concept note per character folder**

Why:

- the master document tells you what the whole folder is doing
- the per-character files explain the exact build lane and current status
- using only one giant file becomes unwieldy
- using only separate files makes the folder lose its identity

So the recommended model is:

- master overview at root
- specific notes in:

## Why This Folder Matters

This folder keeps the NPP honest.

It prevents three different things from being confused:

- standard onboarding pregens
- powerful or unusual concept examples
- active design experimentation

That separation is useful because FRH benefits from ambitious character design, but new players still need a clean front door.
